# IT Trouble Ticket Management System

## Overview
This project is a Java-based IT Trouble Ticket Management System developed to simulate real-world help desk operations. It allows users to create, manage, and resolve support tickets efficiently while demonstrating core Object-Oriented Programming (OOP) concepts.

## Features
- Create and manage IT support tickets
- Assign tickets to technicians or users
- Track ticket status (Open, In Progress, Resolved)
- Update and resolve tickets through a structured workflow
- Console-based interaction for user input and system output

## OOP Concepts Applied
- **Encapsulation** – Organized data and methods within classes
- **Inheritance** – Reused and extended base class functionality
- **Polymorphism** – Enabled flexible method behavior and interactions

## Technologies Used
- Java
- Eclipse IDE

## System Functionality
The system simulates a help desk environment where tickets move through a lifecycle:
1. Ticket creation  
2. Assignment to a technician  
3. Status updates during processing  
4. Final resolution and closure  

## Testing & Debugging
- Tested program functionality using Eclipse IDE
- Debugged logical errors to ensure smooth ticket processing and accurate status tracking

## Project Structure
- `src/` – Contains all Java source files
- Main class – Runs the ticketing system

## How to Run
1. Import the project into Eclipse IDE  
2. Locate and run the main class  
3. Follow the console prompts to interact with the system  

## Purpose
The goal of this project was to apply Java OOP principles to build a functional system that reflects real-world IT support workflows and improves understanding of software design and structure.
